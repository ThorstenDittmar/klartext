# Essence Kernel — Alpha States & Checklists

> **Scope.** Reference: the **seven Essence kernel Alphas** with their full state lifecycles and per-state
> checklists — the operational heart of the Kernel ("done" as a checkable state, not a feeling).
> **Out of scope.** What an Alpha *is* and how the Kernel works (`semat-definition.md`); our current state
> assessments (those live in `continuous-improvement.md` — this file is the measuring stick, not the
> measurement).
> **Anti-pattern guarded.** Dependence on external, form-gated material for day-to-day method work (RC4);
> "done by feeling" instead of checkable state (the H01 mistake).
> **Language.** English — documentation-language rule.

## Customer area

### Opportunity
*The set of circumstances that makes it appropriate to develop or change a software system.*

1. **Identified** — Idea behind opportunity identified · At least one investing stakeholder interested ·
   Other stakeholders identified
2. **Solution Needed** — Solution identified · Stakeholders' needs established · Problems and root causes
   identified · Need for a solution confirmed · At least one solution proposed
3. **Value Established** — Opportunity value quantified · Solution impact understood · System value
   understood · Success criteria clear · Outcomes clear and quantified
4. **Viable** — Solution outlined · Solution possible within constraints · Risks acceptable & manageable ·
   Solution profitable · Reasons to develop solution understood · Pursuit viable
5. **Addressed** — Opportunity addressed · Solution worth deploying · Stakeholders satisfied
6. **Benefit Accrued** — Solution accrues benefits · ROI acceptable

### Stakeholders
*The people, groups, or organizations who affect or are affected by a software system.*

1. **Recognized** — Stakeholder groups identified · Key stakeholder groups represented · Responsibilities
   defined
2. **Represented** — Responsibilities agreed · Representatives authorized · Collaboration approach agreed ·
   Way of working supported & respected
3. **Involved** — Representatives assist the team · Timely feedback and decisions provided · Changes promptly
   communicated
4. **In Agreement** — Minimal expectations agreed · Rep's happy with their involvement · Rep's input valued ·
   Team's input valued · Priorities clear & perspectives balanced
5. **Satisfied for Deployment** — Stakeholder feedback provided · System ready for deployment
6. **Satisfied in Use** — Feedback on system use available · System meets expectations

## Solution area

### Requirements
*What the software system must do to address the opportunity and satisfy the stakeholders.*

1. **Conceived** — Stakeholders agree system is to be produced · Users identified · Funding stakeholders
   identified · Opportunity clear
2. **Bounded** — Development stakeholders identified · System purpose agreed · System success clear · Shared
   solution understanding exists · Requirement's format agreed · Requirements management in place ·
   Prioritization scheme clear · Constraints identified & considered · Assumptions clear
3. **Coherent** — Requirements shared · Requirements' origin clear · Rationale clear · Conflicts addressed ·
   Essential characteristics clear · Key usage scenarios explained · Priorities clear · Impact understood ·
   Team knows & agrees on what to deliver
4. **Acceptable** — Acceptable solution described · Change under control · Value to be realized clear · Clear
   how opportunity addressed · Testable
5. **Addressed** — Enough addressed to be acceptable · Requirements and system match · Value realized clear ·
   System worth making operational
6. **Fulfilled** — Stakeholders accept requirements · No hindering requirements · Requirements fully satisfied

### Software System
*A system made up of software, hardware, and data that provides its primary value by the execution of the
software.*

1. **Architecture Selected** — Architecture selection criteria agreed · HW platforms identified · Technologies
   selected · System boundary known · Decisions on system organization made · Buy, build, reuse decisions
   made · Key technical risks agreed to
2. **Demonstrable** — Key architectural characteristics demonstrated · System exercised & performance
   measured · Critical HW configurations demonstrated · Critical interfaces demonstrated · Integration with
   environment demonstrated · Architecture accepted as fit-for-purpose
3. **Usable** — System can be operated · System functionality tested · System performance acceptable · Defect
   levels acceptable · System fully documented · Release content known · Added value clear
4. **Ready** — User documentation available · System accepted as fit-for-purpose · Stakeholders want the
   system · Operational support in place
5. **Operational** — System available for use · System live · Agreed service levels supported
6. **Retired** — Replaced or discontinued · No longer supported · No authorized users · Updates stopped

## Endeavour area

### Work
*Activity involving mental or physical effort done in order to achieve a result.*

1. **Initiated** — Required result clear · Constraints clear · Funding stakeholders known · Initiator
   identified · Accepting stakeholders known · Source of funding clear · Priority clear
2. **Prepared** — Commitment made · Cost and effort estimated · Resource availability understood · Risk
   exposure understood · Acceptance criteria established · Sufficiently broken down to start · Tasks
   identified and prioritized · Credible plan in place · Funding in place · At least one team member ready ·
   Integration points defined
3. **Started** — Development started · Progress monitored · Definition of done in place · Tasks being
   progressed
4. **Under Control** — Tasks being completed · Unplanned work under control · Risks under control · Estimates
   revised to reflect performance · Progress measured · Re-work under control · Commitments consistently met
5. **Concluded** — Only admin tasks left · Results achieved · Resulting system accepted
6. **Closed** — Lessons learned · Metrics available · Everything archived · Budget reconciled & closed · Team
   released · No outstanding, uncompleted tasks

### Team
*A group of people actively engaged in the development, maintenance, delivery or support of a specific
software system.*

1. **Seeded** — Mission defined · Constraints known and defined · Growth mechanisms in place · Composition
   defined · Responsibilities outlined · Required commitment level clear · Required competencies identified ·
   Size determined · Governance rules defined · Leadership model selected
2. **Formed** — Enough members recruited · Roles understood · How to work understood · Members introduced ·
   Individual responsibilities accepted and aligned to competencies · Members accepting work · External
   collaborators identified · Communication mechanisms defined · Members commit to team
3. **Collaborating** — Works as one unit · Communication open and honest · Focused on mission · Members know
   each other
4. **Performing** — Consistently meeting commitments · Continuously adapting to change · Addresses problems ·
   Rework and backtracking minimized · Waste continuously eliminated
5. **Adjourned** — Responsibilities fulfilled · Members available to other teams · Mission concluded

### Way of Working
*The tailored set of practices and tools used by a team to guide and support their work.*

1. **Principles Established** — Team actively support principles · Stakeholders agree with principles · Tool
   needs agreed · Approach recommended · Operational context understood · Practice & tool constraints known
2. **Foundation Established** — Key practices & tools selected · Practices needed to start work agreed ·
   Non-negotiable practices & tools identified · Gaps between available and needed way of working understood ·
   Gaps in capability understood · Integrated way of working available
3. **In Use** — Practices & tools in use · Regularly inspected · Adapted to context · Supported by team ·
   Feedback mechanisms in place · Practices & tools support collaboration
4. **In Place** — Used by whole team · Accessible to whole team · Inspected and adapted by whole team
5. **Working Well** — Predictable progress being made · Practices naturally applied · Tools naturally support
   way-of-working · Continually tuned
6. **Retired** — No longer in use · Lessons learned shared
