# Method Keeper

> **Essence type:** Pattern
> **Advances Alpha:** Way of Working (the method stays coherent and owned)  ·  **Work Products:** none (a role/discipline applied to method topics)
> **External dependencies (referenced Resources):** none
> **Enforcement:** ritual  ·  **NN:** —
> **Status:** living  ·  **Owner:** OE (owns the method)
> *(Subtype: role pattern — a discipline applied by a role; clarifier kept out of the Essence-type enum value.)*

## Purpose

A way-of-working topic is **located in Essence terms before any solution is proposed** — by the role that
owns the method. Guards the solution-first failure mode: jumping to a fix while talking past the method,
inventing parallel concepts instead of placing the topic in the existing meta-language.

## Definition / delta

The method-keeper role, when a way-of-working topic arises:

1. **Locate before solutioning** — first answer *what kind of thing is this in the method?* Which element
   type (Alpha / Practice / Pattern / Work Product / Activity / Competency)? Which Alpha does it advance?
   This precedes any proposal for how to change it.
2. **Use the meta-language** — frame the topic in the team's Essence vocabulary, not ad-hoc terms, so it
   slots into the Kernel rather than reinventing it.
3. **Owned by the method owner** — the keeper is the role accountable for the method; way-of-working
   changes are framed and routed through that role, not improvised across domains.

Requires **Method Literacy ≥ 2** (*Applies*) for the keeper. Underpins the Classify step of the
**Improvement Step** practice and the Retrospective leadership requirement.

**Completion (Done):** the topic was located in Essence terms (element type + Alpha) **before** any solution
was proposed · framed in the team's meta-language · routed through the method owner.

**Enforcement note (generic).** Ritual; a discipline rather than a mechanism. Composes with
[[kb-first-lookup]] (own reference first when locating) and [[blocker-protocol]] (classification is what
makes an escalation routable).
