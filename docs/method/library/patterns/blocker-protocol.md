# Blocker Protocol

> **Essence type:** Pattern
> **Advances Alpha:** Work (keeps blocked work from drifting Out of Control)  ·  **Work Products:** none (an escalation manoeuvre)
> **External dependencies (referenced Resources):** none
> **Enforcement:** ritual  ·  **NN:** ✓ (on any unlocatable or blocked step)
> **Status:** living  ·  **Owner:** OE (pattern form)

## Purpose

When a step cannot be located in the method or is otherwise blocked, **stop and escalate to the owner
rather than improvise**. Guards the silent-improvisation failure mode: an agent inventing a path around a
blocker, producing work that is unaccountable and outside the method.

## Definition / delta

On hitting an unlocatable or blocked step:

1. **STOP** — do not invent a workaround, do not proceed on a guess.
2. **Classify** — name the blocker in the team's meta-language: which Alpha is stalled, which element type
   is missing, which root-cause class (RC) does it fit? Classification turns a vague "I'm stuck" into a
   locatable, routable problem.
3. **Escalate to the owner** — hand the classified blocker to the accountable owner of that element/domain
   (a briefing to the responsible agent), and wait for direction. The deciding party owns the resolution.

The agent never improvises past a blocker it cannot locate; it makes the blocker visible and routes it.

**Completion (Done):** work stopped at the blocker (no improvised workaround) · blocker classified in
Essence/RC terms · escalated to the named owner with enough context to act · the agent waits for direction
rather than proceeding on a guess.

**Enforcement note (generic).** Ritual; resists mechanization (recognising "I cannot locate this" is a
judgement). First run: Hannibal B2, 2026-06-10. Pairs naturally with [[memory-park]] (park context that
would be lost while blocked) and [[method-keeper]] (locating the topic is what makes classification possible).
