# Improvement Step

> **Essence type:** Practice
> **Advances Alpha:** Way of Working  ·  **Work Products:** decision + rationale at a durable home · updated method-vocabulary entries · routed knowledge briefings (or "none affected") · a "next step" pointer
> **Activity / Activity Space:** provisional — no kernel Activity Space (Way-of-Working-meta); Activity-Space extension deferred per Plan §4.1 → evolve the team's method one decision at a time
> **External dependencies (referenced Resources):** none
> **Enforcement:** ritual  ·  **NN:** ✓
> **Status:** living  ·  **Owner:** OE
> **Enacted as:** — (ritual; skill promotion deferred until a first run validates the shape — "instantiate through use")

## Purpose

Evolve a team's way of working **one decision at a time**, such that **no insight is ever lost** — without
anyone having to remember to capture it. This is the load-bearing brick of a continuous-improvement loop:
each run advances the Way-of-Working Alpha toward *In Use* → *Working Well*.

## Definition / delta

A single Improvement Step proceeds in six steps:

1. **Pick up** — name exactly **one** topic/decision (not several at once).
2. **Classify** — locate the topic in the method's meta-language **before** discussing solutions: which
   element type is it (Alpha / Practice / Pattern / Work Product / Activity / Competency)? Which Alpha does
   it advance? Does the standard already predefine it (**KB-first lookup**: consult the team's own
   method-language reference first, the external standard only on a miss)? This step keeps the team
   *thinking* in the meta-language instead of inventing parallel concepts.
3. **Clarify & decide** — present options + a recommendation; the **decision-owner decides**.
4. **Record** — store the decision at its agreed durable home, **including the rationale (the "why")**.
   This is the capture-first pillar — a decision that lives only in conversation is not recorded.
5. **Route** — inform affected parties of the decision. This is what earns the Way-of-Working state
   *In Use* (the team actually uses the practice).
6. **Set next step** — leave a pointer to the next open topic, so the loop never stalls and a
   context-reset cannot sever it.

**Completion (Done):** exactly one topic addressed · located in meta-language terms + checked against the
standard (KB-first) **before** deciding · a decision was made by the decision-owner · decision **and
rationale** recorded at a durable home (not only in conversation) · affected parties identified and briefed
(or explicitly "none affected") · a next-step pointer exists.

**Enforcement note (generic).** A ritual by nature; it becomes mechanizable only in parts (e.g. a lint that
a "next step" pointer is present). Promotion to a skill/tool should **derive from this card and stay in
sync** — two diverging representations are a second-source-of-truth defect; this card stays the SSOT.
