# Way-of-Working Learnings

> **Scope.** One file per **process / way-of-working learning** — from retrospectives, improvement steps or
> incidents. Filename: `YYYY-MM-DD-<slug>.md`.
> **Out of scope.** QA / test-gap learnings → `docs/superpowers/qa-learnings/`. Decisions and their rationale
> → `continuous-improvement.md`. Improvement tracking → the Improvement Register
> (`continuous-improvement.md` §3).
> **Anti-pattern guarded.** Insights that live only in chat and get lost (the capture pillar; RC1).
> **Language.** English — documentation-language rule.

A learning entry states: what happened (evidence) · what we learned · which root cause it maps to (RC1–RC6,
or a candidate new one) · what changed because of it (link to the Improvement Register entry, if any).
